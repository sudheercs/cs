📖 Introduction to Boolean Algebra

Boolean Algebra is a branch of mathematics that deals with variables that can take only two values:

🔴 Binary Value 0

Represents FALSE, OFF, LOW, or NO

🟢 Binary Value 1

Represents TRUE, ON, HIGH, or YES

Origin: Developed by George Boole in 1854 in his book "An Investigation of the Laws of Thought". It became the mathematical foundation for digital circuit design and modern computing.

Boolean Algebra uses logical operators (AND, OR, NOT) to perform operations on binary variables, forming the basis of all digital electronics and programming logic.

🔢 Boolean Variables & Constants

Boolean Constants

Fixed values in Boolean algebra:

  • 0 → Logical FALSE
  • 1 → Logical TRUE
Constants are the only two possible values in the Boolean domain.

Boolean Variables

Symbols (A, B, C, X, Y, Z...) that can assume either 0 or 1.

  • A = 0 means variable A is FALSE
  • B = 1 means variable B is TRUE
A Boolean expression combines variables and operators to produce a Boolean result.

Example: If A = 1 and B = 0, then (A AND B) = 0

⚙️ Basic Boolean Operations

There are three fundamental operations in Boolean Algebra:

1. AND (Conjunction)

Symbol: · or ∧

Result is TRUE only if all inputs are TRUE.

A · B = AB

2. OR (Disjunction)

Symbol: + or ∨

Result is TRUE if at least one input is TRUE.

A + B

3. NOT (Negation)

Symbol: ‾ or ' or ¬

Inverts the input value.

Ā or A'

These three operations are functionally complete — any Boolean function can be expressed using only AND, OR, and NOT.

📜 Basic Laws of Boolean Algebra

Boolean Algebra follows a set of fundamental laws that allow simplification of expressions:

Identity Law

A + 0 = A

A · 1 = A

Null Law

A + 1 = 1

A · 0 = 0

Idempotent Law

A + A = A

A · A = A

Complement Law

A + Ā = 1

A · Ā = 0

Involution Law

Ā̄ = A

Double negation returns the original value.

🔀 Commutative, Associative & Distributive Laws

🔁 Commutative Law

A + B = B + A

A · B = B · A

Order of variables does not matter.

🔗 Associative Law

(A + B) + C = A + (B + C)

(A · B) · C = A · (B · C)

Grouping of variables does not matter.

📐 Distributive Law

A · (B + C) = AB + AC

A + (B · C) = (A+B)(A+C)

Similar to algebra but with a dual form.

🔥 De Morgan's First Theorem

Theorem 1: The complement of a sum equals the product of the complements.
(A + B)̄ = Ā · B̄

Proof via Truth Table:

ABA+B(A+B)̄ĀB̄Ā · B̄
0001111
0110100
1010010
1110000

Columns (A+B)̄ and Ā · B̄ are identical ✓ — the theorem is proved.

🔥 De Morgan's Second Theorem

Theorem 2: The complement of a product equals the sum of the complements.
(A · B)̄ = Ā + B̄

Proof via Truth Table:

ABA·B(A·B)̄ĀB̄Ā + B̄
0001111
0101101
1001011
1110000

Columns (A·B)̄ and Ā + B̄ are identical ✓ — the theorem is proved.

📐 Other Important Theorems

Absorption Law

A + AB = A

A(A + B) = A

Redundancy Law

A + ĀB = A + B

A(Ā + B) = AB

Consensus Theorem

AB + ĀC + BC = AB + ĀC

The term BC is redundant and can be removed.

Duality Principle

Every Boolean expression remains valid if we swap + with · and 0 with 1.

This creates a dual expression.

🚪 Logic Gates – Introduction

Logic gates are electronic circuits that implement Boolean operations. They are the building blocks of all digital systems.

Basic Gates

  • AND Gate
  • OR Gate

Universal Gates

  • NAND Gate
  • NOR Gate

Any logic circuit can be built using only NAND or only NOR gates.

Special Gates

✂️ Simplification using Boolean Algebra

Boolean expressions can be simplified using laws and theorems to reduce the number of gates in a circuit.

Example: Simplify   F = AB + AB̄ + ĀB

F = AB + AB̄ + ĀB
   = A(B + B̄) + ĀB   ← Distributive Law
   = A(1) + ĀB       ← Complement Law (B+B̄=1)
   = A + ĀB           ← Identity Law
   = A + B           ← Redundancy Law

The original expression with 3 terms & 6 literals simplifies to just A + B (1 OR gate)!

🗺️ Karnaugh Map (K-Map) – Introduction

A Karnaugh Map is a graphical method for simplifying Boolean expressions. It arranges truth table values in a grid so that adjacent cells differ by only one variable.

2-Variable K-Map

B=0B=1
A=0m₀m₁
A=1m₂m₃

Cells: m₀=ĀB̄, m₁=ĀB, m₂=AB̄, m₃=AB

3-Variable K-Map

AB \ CC=0C=1
00m₀m₁
01m₂m₃
11m₆m₇
10m₄m₅

Note: Gray code ordering (00→01→11→10)

Adjacent 1's are grouped in powers of 2 (1,2,4,8...) to find the minimal expression.
