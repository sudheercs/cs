Character Encoding Schemes

ASCII, ISCII, and Unicode (UTF-8 & UTF-32)

Use ← → arrow keys or buttons below to navigate

1. What is Character Encoding?

How computers understand human language.

The Problem

Computers only understand binary numbers (0s and 1s). They do not understand letters like 'A', symbols like '#', or emojis like '😃'.

The Solution

A Character Encoding Scheme is a standardized dictionary that maps a unique numerical value (which can be stored in binary) to every character we want to display.

💡 Example Flow

Keyboard press 'A' ➔ Look up 'A' in the Encoding Standard ➔ Finds numeric value 65 ➔ Sends Binary 01000001 to Memory/CPU.

2. ASCII (American Standard Code for Information Interchange)

The original standard for English characters on the early internet.

Standard 7-bit ASCII

ASCII originally used exactly 7 bits per character. This means it could represent a total of $2^7 = 128$ unique characters.

  • English Alphabets (Upper: A-Z, Lower: a-z)
  • Digits (0-9)
  • Punctuation (!, @, #, $, etc.)
  • Control characters (Carriage Return, Tab)

Important ASCII Values

Character Decimal Value Binary
'A' 65 01000001
'a' 97 01100001
'0' 48 00110000
Space 32 00100000

3. Extended ASCII and Its Limitations

Extended ASCII (8-bit)

Computers organize data into 8-bit bytes. Standard ASCII only used 7 bits, leaving the 8th bit as 0. To use this wasted bit, "Extended ASCII" was created.

It uses all 8 bits ($2^8 = 256$ characters). The additional 128 characters were used for European accented letters (like é, ñ) and drawing symbols.

The Great Limitation

There are over 7,000 spoken languages in the world. Chinese alone has over 50,000 characters.

A 256-character limit is completely insufficient for a globalized world. ASCII cannot represent Arabic, Hindi, Japanese, or any non-Western alphabet.

4. ISCII (Indian Script Code for Information Interchange)

India's solution for digital representation of regional scripts.

What is ISCII?

Developed in the late 1980s, ISCII is an 8-bit encoding scheme standard designed by the Bureau of Indian Standards (BIS).

It was created because standard ASCII had no provision for Indian languages.

How it Works

Since it is 8-bit (256 characters):

  • The first 128 codes (0-127) are identical to standard ASCII (maintaining English compatibility).
  • The next 128 codes (128-255) are used to represent characters from Indian scripts based on the ancient Brahmi script.

Supported Scripts: Devanagari (Hindi, Marathi, Sanskrit), Bengali, Gujarati, Gurmukhi, Odia, Tamil, Telugu, Kannada, and Malayalam.

5. Unicode: The Universal Solution

One standard to rule them all.

🌍

The Goal

The Unicode Standard provides a unique number for every character, no matter what the platform, no matter what the program, no matter what the language.

📈

Capacity

Unlike ASCII (256 max), Unicode can define over 1.1 Million distinct characters. It currently includes English, Hindi, Arabic, ancient Egyptian Hieroglyphs, and all Emojis! 🍕

🔢

Code Points

Unicode doesn't directly dictate how binary is stored; it just assigns an ID called a Code Point. Written as U+XXXX (e.g., U+0041 for 'A', U+1F600 for 😀).

6. Unicode Transformation Formats (UTF)

Code Points must be converted into actual binary Bytes to be stored on a hard drive.

Unicode is purely a conceptual chart. To save a Unicode Code Point (like U+0915 for Devanagari 'क') to memory, we use a specific encoding algorithm called a UTF.

UTF-8

Variable-width encoding (uses 1, 2, 3, or 4 bytes per character). Optimizes memory for English text.

UTF-32

Fixed-width encoding (uses exactly 4 bytes for every character). Wasteful on memory, but fast to process.

7. UTF-8 Encoding (Deep Dive)

The dominant encoding of the World Wide Web (used by ~98% of websites).

Variable Width

UTF-8 is brilliant because it is flexible:

  • Standard English (A-Z) takes 1 Byte.
  • European/Arabic/Hebrew letters take 2 Bytes.
  • Asian scripts (Hindi, Chinese) take 3 Bytes.
  • Emojis and historic scripts take 4 Bytes.

Backward Compatibility

The first 128 characters of UTF-8 use exactly 1 byte and match ASCII perfectly! Any old text file created in the 1980s using standard ASCII is implicitly also a perfectly valid UTF-8 file.

8. UTF-32 Encoding

Fixed width for fast programmatic slicing.

How it works

In UTF-32, every single character takes exactly 32 bits (4 bytes) of memory. Whether it is 'A' or '😀', it occupies 4 bytes.

Pros & Cons

  • ✅ Fast Indexing: If you want to jump to the 100th character in a string, the CPU exactly calculates its memory address (100 × 4 bytes). In UTF-8, you have to scan from the start because widths vary.
  • ❌ Massive Memory Waste: Standard English files become 4 times larger on the hard drive compared to ASCII/UTF-8.

📝 Summary Comparison

Scheme Bits per Char Total Characters Primary Purpose
ASCII (Std) 7 bits 128 Original English encoding.
ASCII (Ext) 8 bits 256 Added European accents.
ISCII 8 bits 256 Indian scripts + English (ASCII compatible).
UTF-8 (Unicode) 8 to 32 bits 1.1 Million+ Global web standard, memory efficient.
UTF-32 (Unicode) 32 bits (Fixed) 1.1 Million+ Global, fast indexing, high memory usage.
