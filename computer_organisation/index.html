1. Introduction to Computer System

What is a computer? Understanding the fundamental concept and its evolution.

🖥️

Definition

A computer is an electronic device that accepts data as input, processes it according to a set of instructions (program), and produces meaningful output (information). It can also store data for future use.

⚙️

IPO Cycle

Every computer operates on the Input → Process → Output (IPO) cycle. Data enters via input devices, the CPU processes it using stored instructions, and results are delivered through output devices.

📐

Characteristics

  • Speed — Millions of calculations per second
  • Accuracy — Error-free if input is correct (GIGO)
  • Diligence — No fatigue or boredom
  • Versatility — Multi-purpose machine
  • Storage — Vast data storage capacity
  • Automation — Minimal human intervention
📜

Generations Overview

  • 1st (1940–56) — Vacuum tubes, ENIAC
  • 2nd (1956–63) — Transistors
  • 3rd (1964–71) — Integrated Circuits
  • 4th (1971–present) — Microprocessors
  • 5th (emerging) — AI & quantum computing

💡 Real-World Example

When you type a search query into Google, your keyboard (input) sends data to the CPU (processing), which communicates with servers. The results appear on your monitor (output). The entire IPO cycle completes in milliseconds.

2. Hardware

The physical, tangible components of a computer system.

🔩

What is Hardware?

Hardware refers to all the physical parts of a computer that you can see and touch. Without hardware, software has no platform to run on. Hardware provides the computational power, storage, and I/O capabilities.

🏗️

Categories of Hardware

  • Input Devices — Keyboard, Mouse, Scanner
  • Output Devices — Monitor, Printer, Speaker
  • Processing Unit — CPU (ALU + CU + Registers)
  • Memory/Storage — RAM, ROM, HDD, SSD
  • Motherboard — Main circuit board connecting all
  • Communication — NIC, Modem, Wi-Fi adapter

💡 Example — Inside a Desktop PC

Opening a desktop reveals the motherboard hosting the CPU, RAM sticks, and expansion slots. A power supply unit (PSU) provides electricity. An HDD/SSD stores the operating system and files. Cooling fans prevent overheating. A GPU (Graphics Processing Unit) handles display rendering.

3. Software

The intangible instructions that make hardware perform tasks.

📦

System Software

Manages hardware resources and provides a platform for application software.

  • Operating Systems — Windows, Linux, macOS
  • Device Drivers — Printer driver, GPU driver
  • Utilities — Disk Defragmenter, Antivirus
  • Language Translators — Compiler, Interpreter, Assembler
📱

Application Software

Designed for end-users to perform specific tasks.

  • General Purpose — MS Word, Excel, Chrome
  • Specific Purpose — Tally (Accounting), AutoCAD
  • Customised — Railway reservation system

💡 Relationship Example

When you click a file on your desktop, the OS (system software) interprets the mouse click (via the mouse driver), locates the file on the HDD (hardware), and opens it with the appropriate application (e.g., Adobe Reader for PDFs). Hardware and software are interdependent.

4. Input Devices

Devices used to enter data and instructions into a computer.

⌨️

Keyboard

The most common input device with 104–108 keys. Types: QWERTY, DVORAK, ergonomic. Keys include alphanumeric, function (F1–F12), cursor control, and modifier keys (Ctrl, Alt, Shift).

🖱️

Mouse

A pointing device that controls the cursor. Types: Mechanical (ball-based), Optical (LED sensor), Wireless (Bluetooth/RF). Operations: click, double-click, drag, scroll.

📷

Scanner & Camera

Scanner — Converts physical documents to digital images (flatbed, handheld, drum). Webcam / Digital Camera — Captures images and video for input.

🎤

Microphone & Others

Microphone — Audio input for voice commands, recording. Touchscreen — Dual I/O (input + output). Barcode Reader — scans product barcodes. Biometric — fingerprint, retina scanner. Joystick — gaming input.

💡 Example — Supermarket Checkout

A barcode reader scans the product, a weighing scale (sensor input) measures produce, and a touchscreen POS registers the payment. Multiple input devices work together in real-world systems.

5. Output Devices

Devices that convey processed information to the user.

🖥️

Monitor (Visual Display Unit)

Displays text, images, and video. Types: CRT (old, bulky), LCD (Liquid Crystal Display), LED (backlit LCD), OLED (organic, per-pixel lighting). Resolution measured in pixels (e.g., 1920×1080 = Full HD).

🖨️

Printer

Impact — Dot-matrix (uses pins striking ribbon). Non-impact — Inkjet (sprays tiny droplets), Laser (uses toner & heat, fastest for text). 3D Printer — Creates physical objects layer by layer.

🔊

Speaker & Headphones

Convert digital audio signals to analog sound waves. Used for music, video conferences, voice assistants. Quality measured in frequency response (Hz) and wattage.

📽️

Projector & Plotter

Projector — Projects display onto a large screen (LCD, DLP, laser). Plotter — Prints large-scale engineering drawings, maps, and architectural plans with high precision.

💡 Softcopy vs Hardcopy

Softcopy — Output displayed on screens (temporary, editable). Hardcopy — Output printed on paper (permanent, not directly editable). A PDF on a monitor is softcopy; the same PDF printed is hardcopy.

6. Central Processing Unit (CPU)

The "brain" of the computer — where all processing happens.

🧮 Arithmetic Logic Unit (ALU)

Performs all arithmetic operations (addition, subtraction, multiplication, division) and logical operations (AND, OR, NOT, comparisons). The ALU is where actual data manipulation occurs.

🎛️ Control Unit (CU)

Acts as the coordinator/director. It does not process data itself but directs the flow of data between the CPU, memory, and I/O devices. It fetches instructions from memory, decodes them, and orchestrates execution.

📋 Registers

Ultra-fast, small-capacity storage inside the CPU. Types:

  • Accumulator (ACC) — Stores intermediate results
  • Program Counter (PC) — Address of next instruction
  • Instruction Register (IR) — Current instruction
  • Memory Address Register (MAR)
  • Memory Data Register (MDR)

🔄 Fetch-Decode-Execute Cycle

  • Fetch — CU fetches the next instruction from RAM using the address in the PC
  • Decode — CU decodes the instruction to understand what operation to perform
  • Execute — ALU performs the operation; result stored in register or memory
  • Repeat — PC increments; cycle continues billions of times per second

💡 Example — Adding Two Numbers

Instruction: ADD R1, R2, R3 (Add contents of R2 and R3, store in R1)

1. Fetch: CU reads instruction from memory

2. Decode: CU identifies it as an ADD operation

3. Execute: ALU adds values in R2 and R3, result → R1

⏱️ Clock Speed

Measured in GHz (Gigahertz). A 3.5 GHz CPU performs 3.5 billion clock cycles per second. Higher clock speed ≈ faster processing (but architecture matters too).

8. Primary Memory (Main Memory)

The memory directly accessible by the CPU — RAM and ROM.

⚡

RAM (Random Access Memory)

Volatile — data erased when power is turned off. Used to store currently running programs and data being actively processed.

  • SRAM (Static RAM) — Uses flip-flops, faster, used in cache. No need to refresh. Expensive.
  • DRAM (Dynamic RAM) — Uses capacitors, needs periodic refresh, cheaper, used as main memory.
  • SDRAM — Synchronized with CPU clock
  • DDR SDRAM — DDR4, DDR5 (latest) — Double Data Rate

💡 Example

When you open Chrome, the program is loaded from SSD/HDD → RAM. Each new tab consumes more RAM (often 100–300 MB each). When RAM fills up, the system slows due to swapping (using HDD as virtual memory).

🔒

ROM (Read-Only Memory)

Non-volatile — data persists without power. Contains firmware and boot instructions (BIOS/UEFI).

  • PROM — Programmable once (fuse-based)
  • EPROM — Erasable using UV light
  • EEPROM — Electrically erasable, sector-by-sector
  • Flash Memory — Type of EEPROM, used in USB drives, SSDs

9. Cache Memory

A small, ultra-fast memory between the CPU and RAM that bridges the speed gap.

⚡

What is Cache?

Cache memory is a high-speed SRAM that stores frequently accessed data and instructions. It exploits the principle of locality of reference — programs tend to access the same data (temporal) or nearby data (spatial) repeatedly.

📊

Levels of Cache

  • L1 Cache — Smallest (32-64 KB per core), fastest, built into CPU core
  • L2 Cache — Larger (256 KB–1 MB per core), slightly slower, on-die
  • L3 Cache — Shared among all cores (4-64 MB), slowest of the three
🎯

Cache Hit vs Miss

Cache Hit — Requested data IS found in cache → fast access.
Cache Miss — Data NOT found → must fetch from slower RAM.
Hit Ratio = Hits / (Hits + Misses). Modern CPUs aim for >95% hit ratio.

💡 Real-World Analogy

Think of cache like a desk vs a bookshelf vs a library. You keep the book you're currently reading (L1) on your desk. Recently used books (L2) sit on a nearby shelf. Older references (L3) go to the bookshelf. The library (RAM) has everything but takes much longer to access.

10. Secondary Memory (Auxiliary Storage)

Non-volatile, large-capacity storage for permanent data retention.

💿

Hard Disk Drive (HDD)

Uses magnetic storage on spinning platters. Read/write heads float on an air cushion. Measured in RPM (5400/7200/10000). Capacity: typically 500 GB – 20 TB. Cheaper per GB but slower and has moving parts (prone to physical damage).

⚡

Solid State Drive (SSD)

Uses flash memory (NAND chips) — no moving parts. 5–10× faster than HDD. Types: SATA SSD (~550 MB/s), NVMe SSD (~3500-7000 MB/s via PCIe). More durable, shock-resistant. Higher cost per GB.

💾

Removable Media

  • USB Flash Drive (Pen Drive) — Portable, 4 GB – 2 TB
  • SD Card — Used in cameras, phones
  • Optical Discs — CD (700 MB), DVD (4.7/8.5 GB), Blu-ray (25-128 GB)
  • Magnetic Tape — Sequential access, used for enterprise backups (up to 45 TB per cartridge)
☁️

Cloud Storage

Data stored on remote servers accessed via the internet. Examples: Google Drive, AWS S3, Dropbox. Advantages: accessible anywhere, scalable, auto-backup. Concerns: internet dependency, privacy, ongoing cost.

💡 HDD vs SSD — Boot Time Comparison

A Windows PC with HDD boots in ~45–90 seconds. The same PC with an NVMe SSD boots in ~8–15 seconds. SSDs excel at random read/write (4K) operations, making the entire OS feel snappier.

🔢 Binary vs Decimal

Binary (IEC): 1 KiB = 2¹⁰ = 1024 bytes (used by OS)

Decimal (SI): 1 kB = 10³ = 1000 bytes (used by manufacturers)

This is why a "500 GB" HDD shows as ~465 GB in Windows: 500,000,000,000 ÷ 1024³ ≈ 465 GiB.

💡 Quick Calculation Example

Q: How many bits in 2 KB?

A: 2 KB = 2 × 1024 Bytes = 2048 Bytes

2048 × 8 bits = 16,384 bits

Q: Convert 5 GB to MB

A: 5 × 1024 = 5120 MB

📝 Summary — Key Takeaways

Quick revision of all major concepts covered.

🖥️ Computer System

An electronic device following the IPO cycle. Characteristics: speed, accuracy, diligence, versatility, storage, automation.

🔩 Hardware

Tangible components: Input, Output, CPU, Memory, Motherboard. Without hardware, software cannot function.

📦 Software

Intangible programs: System software (OS, drivers) and Application software (Word, Chrome). Interdependent with hardware.

⌨️ Input / 🖥️ Output

Input: Keyboard, Mouse, Scanner, Mic. Output: Monitor, Printer, Speaker. Some devices are dual (touchscreen).

🧠 CPU

Brain of the computer — ALU (arithmetic/logic), CU (control/coordination), Registers (ultra-fast storage). Fetch-Decode-Execute cycle.

💾 Memory

Hierarchy: Registers → Cache (L1/L2/L3) → RAM → SSD/HDD. Trade-off between speed, size, and cost. Volatile vs Non-volatile.

🔑 Master Formula

1 Byte = 8 bits → 1 KB = 1024 B → 1 MB = 1024 KB → 1 GB = 1024 MB → 1 TB = 1024 GB → 1 PB = 1024 TB
