Expressions vs. Statements

The two fundamental building blocks of code.

2. What is an Expression?

If it can be "boiled down" to a single value, it's an expression.

An Expression is any combination of values, variables, and operators that the interpreter evaluates to produce a single value.

Examples of Expressions

  • 10 (A literal is an expression evaluating to itself)
  • "Hello" + "World"
  • 5 + (3 * 2)
  • math.sqrt(16)
  • x > 10 (Evaluates to True/False)

The Test

If you can print it via print( ... ), it is an Expression.

You can write print( 5 + 3 ) because 5 + 3 reduces to 8.

3. What is a Statement?

A statement is an instruction that DOES something.

A Statement is an executable command/instruction that the Python interpreter executes. Statements do NOT evaluate to a value.

Examples of Statements

  • x = 10 (Assignment statement)
  • if x > 5: (If statement)
  • for i in range(5): (For statement)
  • import random (Import statement)

The Test

If it causes an error when placed inside a print(), it's a Statement.

You CANNOT write print( x = 10 ). Assignment doesn't return a value.

4. Anatomy of a Line of Code

Often, a Statement CONTAINS an Expression inside of it!

total = price * quantity

Assignment Statement

The entire line is an instruction telling the computer to allocate memory and save data.

Expression

The Right-Hand Side boils down into a single numeric value before the assignment can happen.

5. Quick Check: Expression or Statement?

Code Type Why?
len("Apple") Expression Evaluates to the integer 5.
def my_func(): Statement Instruction to define a function. Does not return a value instantly.
15 > 10 and True Expression Evaluates to the boolean True.
pass Statement A null operation instruction. Does nothing, but is a statement.
name[0:3] Expression Evaluates to a sliced String.

Evaluation of an Expression

The Precedence Rules of Python.

6. How Python Evaluates

When Python encounters a complex expression, it doesn't just read left-to-right like a human reading a book. It follows strict mathematical precedence rules (BODMAS/PEMDAS on steroids).

The Order (Simplified)

  1. ( ) Parentheses
  2. ** Exponentiation
  3. +, - Unary Pos/Neg Signs
  4. *, /, //, % Mult & Div
  5. +, - Add & Subtract
  6. <, >, ==, != Relational
  7. not, and, or Logical

Associativity

If two operators share the same precedence (e.g., * and /), Python evaluates Left-to-Right.

The only exception is ** (Exponentiation), which evaluates Right-to-Left.

7. Evaluation Trace: Example 1

result = 10 + 20 * 3 / 2 - 5

Step-by-Step Breakdown:

  • 1. Identify highest precedence: * and /. L-to-R associative.
  • 2. Evaluate multiplication: 20 * 3 = 60.
    • Expression reduces to: 10 + 60 / 2 - 5
  • 3. Evaluate division: 60 / 2 = 30.0 (True division always yields float).
    • Expression reduces to: 10 + 30.0 - 5
  • 4. Identify next precedence: + and -. L-to-R associative.
  • 5. Evaluate addition: 10 + 30.0 = 40.0.
    • Expression reduces to: 40.0 - 5
  • 6. Evaluate subtraction: 35.0.

8. Evaluation Trace: Example 2 (Mixed Types)

ans = 2 ** 3 ** 2 > 100 and "A" == "a"

Step-by-Step Breakdown:

  • 1. Exponentiation is highest. Remember: Right-to-Left!
  • 2. Evaluate 3 ** 2 = 9.
    • Expression reduces to: 2 ** 9 > 100 and "A" == "a"
  • 3. Evaluate 2 ** 9 = 512.
    • Expression reduces to: 512 > 100 and "A" == "a"
  • 4. Relational Operators (>, ==) evaluate next.
  • 5. 512 > 100 is True. "A" == "a" is False (ASCII).
    • Expression reduces to: True and False
  • 6. Logical and evaluates to False.

9. Short-Circuit Evaluation

Logical OR

If Python sees X or Y, and evaluates X to be True, it immediately stops processing and returns True.

Why? Because only one side needs to be True for an OR statement to succeed. Evaluating the right side is a waste of CPU power.

Logical AND

If Python sees X and Y, and evaluates X to be False, it immediately stops and returns False.

Why? Because an AND statement demands BOTH sides be true. If the first fails, the entire statement is doomed regardless of Y.

# The division by zero NEVER executes, preventing a crash!
safe = (10 > 5) or (50 / 0 == 5)

Type Conversion

Transforming data from one format to another.

10. Why do we need Type Conversion?

In Python, you cannot freely mix certain incompatible data types in an expression. Attempting to do so results in a TypeError.

The Error

age = 18
message = "I am " + age
print(message)

TypeError: can only concatenate str (not "int") to str

The Solution

We must convert the Integer 18 into a String "18" before the + operator can stitch them together.

11. Implicit vs. Explicit Conversion

The two ways types are changed.

1. Implicit Conversion (Coercion)

Python automatically converts smaller/narrower data types into larger/wider data types during evaluation to prevent data loss. The programmer does nothing.

2. Explicit Conversion (Type Casting)

The programmer forces the conversion by wrapping the expression in a built-in function like int(), float(), or str().

12. Implicit Type Conversion Rule

Integers and Floats

If you mix an integer and a float in a mathematical expression, Python automatically upgrades the Integer into a Float to guarantee that no decimal data is lost in the calculation.

x = 10 # int
y = 5.5 # float

ans = x + y
print(ans)
print(type(ans))

Behind the Scenes:

Python internally changes 10 to 10.0.

10.0 + 5.5 = 15.5

Resulting Type: <class 'float'>

13. Explicit Type Casting Functions

Most basic Python data types have a corresponding built-in function to cast other values into their mould.

Function What it does Example
int() Converts to integer (chops decimals) int(5.9) ➔ 5
float() Converts to float (adds .0) float(7) ➔ 7.0
str() Converts to a String text sequence str(18) ➔ "18"
bool() Converts to True or False bool(0) ➔ False

14. Dangers of Explicit Type Casting

Data Loss with int()

Casting a float to an int truncates the decimal completely. It does NOT round.

pi = 3.14159
fake_pi = int(pi)
print(fake_pi) # Output: 3

(All that precision is gone forever).

ValueError with int()

You cannot cast a string to an integer if the string doesn't contain valid base-10 numbers.

word = "Hello"
bad = int(word)
# ValueError: invalid literal for int()

15. Solving the TypeError!

Going back to Slide 14...

To safely concatenate an integer into a sentence, pass the integer directly into the str() casting function within the expression!

age = 18
message = "I am " + str(age) + " years old."
print(message)

Output: I am 18 years old.

Input & Output

Talking to the User.

16. Standard Output: print()

The print() function is used to send data to the standard output device (usually the screen/console).

Separating Items

You can print multiple items by separating them with commas. Python automatically inserts a space between them!

x = "Apple"
y = 10
print(x, y) # Output: Apple 10

End vs Sep Arguments

Advanced parameters exist to change print behavior:

# sep changes the space to a hyphen
print(x, y, sep="-") # Output: Apple-10

# end prevents the newline
print("Hi", end="")

17. Formatted Strings (F-Strings)

The modern way to format output.

Instead of aggressively using str() casting and + signs to stitch variables together, Python 3.6+ introduced f-strings.

Simply place an f before the quote, and inject variables inside curly braces {}.

The Old/Ugly Way

age = 15
sub = "CS"
print("I am " + str(age) + " taking " + sub)

The F-String Way

age = 15
sub = "CS"
print(f"I am {age} taking {sub}")

18. Standard Input: input()

The input() function does three things simultaneously:

  1. Prints an optional prompt string to the screen.
  2. Pauses the entire program and waits until the user presses the Enter key.
  3. Reads everything the user typed and returns it as an Expression!
# 'name' is the variable where the data gets stored.
name = input("Please enter your name: ")
print(f"Welcome back, {name}!")

19. THE GOLDEN INPUT RULE

input() always returns a String.

It does not matter if the user types '10', 'True', or '5.5'. Python will capture it as '"10"', '"True"', and '"5.5"'.

20. The Input / Math Bug

If you ask the user for two numbers, and blindly add them together, it will trigger String Concatenation, not Mathematical Addition!

x = input("Enter first num: ")
# User types 5

y = input("Enter second num: ")
# User types 10

ans = x + y
print(ans)

Output:

510

"5" + "10" = "510". Python stitched the text together.

21. The Solution: Immediate Type Casting

We combine the input() statement and the int() explicit type casting function into a single, beautiful line of code to instantly convert the typed string into a workable number.

# Take the result of input(), and feed it directly into int()
x = int( input("Enter first num: ") )
y = int( input("Enter second num: ") )

ans = x + y
print(f"The final answer is: {ans}")

Application Examples

Bringing it all together.

22. Example 1: Calculating Circle Area

Write a script that asks the user for a radius, and outputs the area of the circle formatted cleanly.

# 1. Input (Must use float() since radius could be a decimal like 2.5)
radius = float(input("Enter circle radius: "))

# 2. Expression Evaluation
pi = 3.14159
area = pi * (radius ** 2) # Exponentiation executes before Mult!

# 3. Formatted Output
print(f"For radius {radius}, the total area is {area}")

23. Example 2: The Bill Splitter

Ask for the total bill quantity, how many friends to split among, and display the exact breakdown rounding to integer values.

total_bill = float(input("Total Bill Amount: ₹"))
friends = int(input("How many people? "))

# Implicit Conversion happens here! Float / Int yields Float.
# Then we explicitly cast the final float answer to int to drop decimals!
split_share = int(total_bill / friends)

print(f"Each friend must pay exactly ₹{split_share}")

24. Special Challenge Issue: "010" is not Base-10

Wait, what if they submit blank space?

If a user strikes 'Enter' without typing, input() returns an empty string "". Feeding an empty string into int() crashes the program with a ValueError.

Developers must use Logical Conditionals (If/Else) or Try/Except blocks to validate user input before mathematically mutating it! (Covered in future units).

25. Overall Summary

Expressions vs Statements

Expressions evaluate to values. Statements are instructions that act upon values or control flow.

Precedence

Python obeys strict order of operations (BODMAS), handling exponents first, then mult/div, then add/sub, then logic.

Type Casting

Implicit coercion (Int>Float) happens automatically. Explicit casting (int(), str()) is forced by you.

The Input Rule

input() is deeply flawed for math unless wrapped instantly in int() or float() because it ALWAYS returns strings.
