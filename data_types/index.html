1. What is a Data Type?

Categorizing the information inside a computer.

Definition

A Data Type specifies the exact classification or type of mathematical, logical, or textual data that a variable holds.

Why does it matter?

The data type tells the Python compiler exactly what operations can be performed on that data.

Example: You can mathematically divide two Numbers, but you cannot mathematically divide two Strings of Text.

2. Determining Data Types

How to ask Python what type a variable is.

The type() function

Python has a built-in function called type() that instantly returns the class classification of any variable or literal.

>>> x = 10
>>> type(x)
<class 'int'>

>>> type("Hello")
<class 'str'>

3. Number Data Types

Used to store numeric values.

In Python, numbers are created instantly when you assign a numeric literal to a variable. Python natively supports three distinct numerical types:

Integer

Whole numbers.
int

Floating Point

Decimal numbers.
float

Complex

Real + Imaginary parts.
complex

4. Integers (int)

Properties

An Integer is a whole number, positive or negative, without any decimals, of unlimited length.

Because Python automatically handles memory, an int can be as massive as your computer's RAM allows.

x = 1
y = -456789
z = 999999999999999999

print(type(y))
# Output: <class 'int'>

5. Floating Point (float)

Properties

A Float (floating point number) is a positive or negative number containing one or more decimals.

Scientific notation using 'e' or 'E' (to indicate power of 10) is also inherently parsed as a float.

x = 1.10
y = -35.59

# Scientific Notation:
z = 35e3 # 35000.0

print(type(z))
# Output: <class 'float'>

6. Complex Numbers (complex)

Properties

A Complex Number contains a real part and an imaginary part. In Python, the imaginary part is written with a `j` immediately following the number (unlike mathematics which uses `i`).

Mainly used in heavy scientific or electrical engineering computations.

x = 3+5j
y = -5j

print(x.real) # Output: 3.0
print(x.imag) # Output: 5.0

print(type(x))
# Output: <class 'complex'>

7. Boolean (bool)

The purest form of logic.

A Boolean represents one of exactly two possible values: True or False. They are vital for conditional statements (like IF/ELSE).

is_active = True
is_admin = False

print(type(is_active))
# Output: <class 'bool'>

Generated implicitly:

Booleans are almost always the result of comparisons:

10 > 9 ➔ True
10 == 9 ➔ False

8. The None Type

How do you define "Nothing"?

NoneType

If you want a variable to exist, but have literally zero data tied to it right now, you assign it None.

None is not 0 (which is an int). None is not "" (which is an empty string). It is its own distinct data type indicating absence.

winner = None

print(type(winner))
# Output: <class 'NoneType'>

9. Concept of Mutability

Mutation = Change

In Python (and computer science at large), every data type falls directly into one of two behavior categories:

Mutable or Immutable.

This dictates whether the underlying memory data can be permanently altered after creation.

10. Immutable Data Types

Definition

Immutable means the state of the object cannot be modified after it is created in memory.

If you try to alter an immutable object, Python destroys the old object in memory and physically creates a brand new one to hold the new value.

Which Types are Immutable?

  • Integers
  • Floats
  • Booleans
  • Strings
  • Tuples

11. Mutable Data Types

Definition

Mutable means the internal data can be updated, appended, or deleted in-place, without changing the original memory address.

Which Types are Mutable?

  • Lists
  • Dictionaries (Mappings)
  • Sets

12. Mutability Analogy

Immutable: A Carved Stone

If you carve "Data" into a piece of solid stone, you cannot erase or change a single letter. To change the text, you must throw away the entire stone block and buy a brand new one to carve on.

Mutable: A Whiteboard

If you write "Data" on a whiteboard, you can easily take an eraser, wipe out letters, and add new ones continuously, while still using the exact same physical whiteboard.

Sequence Data Types

Strings, Lists, and Tuples.

13. The Concept of a 'Sequence'

Ordered Collections

A Sequence data type represents an ordered, continuous collection of similar or different data types. Sequences guarantee that the data is stored in the exact same order it was created.

Because they are perfectly ordered, every single item gets a unique mathematical index (like 0, 1, 2, 3), allowing for instant retrieval.

14. Sequence 1: Strings (str)

Definition

A String is a sequence of characters surrounded by quotes (single, double, or triple).

In Python, a string is mathematically treated as a list of independent characters. Yes, Strings are IMMUTABLE.

s1 = 'Hello'
s2 = "World"

# Multiline
s3 = """This spans
multiple rows"""

print(type(s1))
# <class 'str'>

15. String Indexing

Retrieving data from a sequence via bracket [] notation.

Because strings are ordered sequences, every character gets a numbered position (Index), starting from exactly 0.

0 1 2 3 4 5
P Y T H O N
-6 -5 -4 -3 -2 -1

Python also uniquely supports Negative Indexing, reading from Right-to-Left, starting at -1.

16. Sequence 2: Lists (list)

Definition

A List is Python's most flexible array/sequence. Elements are enclosed in Square Brackets [ ] and separated by commas.

Lists are MUTABLE.

# Creating a list of numbers
marks = [85, 92, 78, 99]

# Retrieving via index 0
print(marks[0])
# Output: 85

17. List Flexibility (Mutable + Mixed)

Changing Data In-Place

Because Lists are mutable, you can re-assign data at specific indexes without destroying the list.

Lists can also hold mixed data types simultaneously (ints, strings, and floats together).

# Mixed data types
data = [10, "Apple", 3.14]

# Overwriting index 1
data[1] = "Banana"

print(data)
# Output: [10, "Banana", 3.14]

18. Sequence 3: Tuples (tuple)

Definition

A Tuple is identical to a List, except elements are enclosed in Parentheses ( ).

Tuples are IMMUTABLE.

# Creating a tuple
rgb = (255, 128, 0)

# Retrieval is identical
print(rgb[1])
# Output: 128

19. Why use Tuples over Lists?

If Tuples are just Lists that you can't edit... why do they exist?

Data Integrity

When you have critical data that should NEVER change natively throughout your program (e.g., GPS Coordinates, Database settings), using a Tuple prevents logical errors where an engineer accidentally overwrites index [0].

Memory Speed

Because Python knows a tuple's size will never change, it optimizes the memory footprint. Iterating through a massive Tuple is mathematically faster than a massive List.

Mapping Data Types

Dictionaries (dict).

20. Dictionaries (dict)

Unordered Key-Value pairs.

Definition

A Dictionary holds data in Key-Value pairs, enclosed in Curly Braces { }.

Unlike sequences, dictionaries have no numerical order. Instead of asking for "Item at Index 0", you ask for "Item tied to Key 'Name'".

student = {
  "name": "Rohan",
  "age": 16,
  "grade": "A"
}

print(student["name"])
# Output: Rohan

21. Properties of Dictionaries

Mutable

Dictionaries are mutable. You can freely add new keys, delete existing keys, or modify the values tied to them.

Key Rules

While the Values can be mutated and be of any type, the Keys must be strictly Immutable Data Types (like Strings, Integers, or Tuples).
