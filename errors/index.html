Python Errors & Exceptions

Understanding Syntax, Logical, and Run-time Errors.

Use ← → arrow keys or buttons below to navigate

1. The Nature of Bugs

In programming, a "bug" is an error or flaw in the code that causes an unexpected result or outright crash.

Errors in Python fall into three distinct categories based on WHEN and WHY they occur.

Syntax Errors

Grammar mistakes. Fails before the program even starts.

Run-time Errors

Execution crashes. Fails while the program is actively running.

Logical Errors

Human mistakes. Valid code, but mathematically/logically creates the wrong answer.

1. Syntax Errors

Breaking the Rules of Grammar.

2. What is a Syntax Error?

Syntax errors occur when you violate the fundamental grammatical rules of the Python language.

The Python Interpreter parses (reads) the entire file before it executes a single line. If it finds a syntax error, it immediately aborts. Zero lines of code are executed.

3. Syntax Example 1: Missing Colons

Control structures like if, for, and while loops MUST end with a colon (:) before the indented block begins.

score = 95

# Missing the colon at the end of the if statement!
if score > 90
    print("Grade A") SyntaxError: expected ':'

4. Syntax Example 2: Unmatched Parentheses/Quotes

Forgetting to close a bracket, parenthesis, or quotation mark prevents the interpreter from understanding where an expression terminates.

# Missing the closing parenthesis!
print("Hello, World!"

# Starting with double quotes but closing with nothing/single!
name = "James SyntaxError: unexpected EOF while parsing

5. Syntax Example 3: Improper Indentation

Unlike languages that use curly braces {}, Python strict uses whitespace indentation to define code blocks. Incorrectly grouping code triggers a specialized Syntax error called an IndentationError.

for i in range(5):
print(i) # This line MUST be indented inward! IndentationError: expected an indented block

2. Run-Time Errors

Crashing during execution.

6. What is a Run-time Error?

Run-time errors (often called Exceptions) occur when the Python grammar is perfectly correct, so the interpreter successfully starts the program.

However, during execution, the machine is asked to perform an illegal or impossible operation (like dividing by zero, or opening a non-existent file), causing a crash.

7. Run-time Example 1: TypeMismatch / ValueError

As we learned in the I/O unit, trying to surgically force data into an incompatible type will crash the running program. This often happens natively with bad user input.

# Program successfully starts!
print("Welcome to the integer machine!")

# What if the user types the word "Apple"?
age = int(input("Enter your age: ")) ValueError: invalid literal for int() with base 10: 'Apple'

8. Run-time Example 2: ZeroDivisionError

Mathematical impossibility. Computer architecture literally cannot divide by zero without returning undefined infinity.

total_money = 5000
friends = 0

# You cannot split cash amongst zero people.
share = total_money / friends ZeroDivisionError: division by zero

9. Run-time Example 3: NameError

A NameError triggers when you attempt to use a variable or function that has not been defined in memory yet.

username = "admin"

# Typo! The variable is 'username', not 'user_name'!
print("Welcome, " + user_name) NameError: name 'user_name' is not defined

3. Logical Errors

The hardest bugs to fix.

10. What is a Logical Error?

A Logical Error (or Semantic Error) occurs when the program runs completely fine without crashing, but it produces the wrong output or behavior.

The computer did exactly what you told it to do. You just told it the wrong thing. There are NO error messages generated by Python for logical errors.

11. Logical Example 1: Wrong Operator

Using addition instead of multiplication, or division instead of floor division.

# GOAL: Calculate the Area of a Rectangle (Area = Length * Width)
length = 10
width = 5

# BUG: The programmer accidentally typed a + instead of a *
area = length + width
print(f"The area is {area}")

# OUTPUT: The area is 15. --> The computer didn't crash, but 10*5 is 50, not 15!

12. Logical Example 2: Operator Precedence Failure

Forgetting BODMAS rules and evaluating in the wrong order.

# GOAL: Calculate the average of 3 grades
math = 90
science = 80
english = 100

# BUG: Division happens BEFORE addition! This only divides english/3.
average = math + science + english / 3
print(average)
# Output: 203.33 --> An average of 3 numbers out of 100 shouldn't be over 200...

# FIX: average = (math + science + english) / 3

13. Logical Example 3: Loop Infinite Offset (Off-by-One)

Using a Relational Operator incorrectly (e.g. > instead of >=), severely altering the bounds of a loop or conditional logic gate.

# GOAL: Wait until you are an adult before granting a license.
# In India, you become an adult natively at 18.
age = 18

# BUG: They used strictly Greater Than, rather than >=
if age > 18:
    print("License Granted")
else:
    print("Denied!")
# Output: Denied! --> (Which is legally incorrect since they are 18).
