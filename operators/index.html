1. What is an Operator?

Definition

An Operator is a special symbol (or keyword) that performs mathematical, relational, or logical computation on one or more variables or values.

The values that the operator acts upon are called Operands.

10 + 20

Here, 10 and 20 are the Operands, and + is the Operator.

4. Basic Math (+, -, *)

Addition & Subtraction

Works exactly as expected with integers and floats.

x = 10.5 + 4
print(x) # Output: 14.5

Multiplication

Also acts as a repetition operator for Strings!

# Numeric Math
x = 5 * 3 # Ouput: 15

5. The Dual Divisions ( / vs // )

True Division ( / )

A single slash always mathematically divides the numbers and GUARANTEES a float output, even if it divides perfectly evenly.

x = 10 / 2
print(x) # Output: 5.0

y = 10 / 3
print(y) # Output: 3.3333333333333335

Floor Division ( // )

Double slashes divide and entirely discard the decimal remainder, cutting it down to the "floor" integer.

x = 10 // 2
print(x) # Output: 5

y = 10 // 3
print(y) # Output: 3

6. Floor Division Danger (Negative Numbers)

Rounding DOWN, not truncating!

The Floor Division operator // does not just chop off the decimal part. It mathematically rounds down to the next lowest infinite integer.

Positive Floor

x = 10 // 3
print(x) # Output: 3

10/3 = 3.33. The integer "below" 3.33 is 3.

Negative Floor

x = -10 // 3
print(x) # Output: -4

-10/3 = -3.33. The integer "below" -3.33 is -4.

7. The Modulus Operator ( % )

Finding the remainder.

How it works

The modulo operator performs division, but instead of giving you the quotient, it hands you back the strict remainder.

If X divides perfectly evenly into Y, the modulus is 0.

# 10 divided by 3 is 3... with 1 remaining
ans1 = 10 % 3
print(ans1) # Output: 1

# 10 divided by 2 is 5... with 0 remaining
ans2 = 10 % 2
print(ans2) # Output: 0

8. Practical Use of Modulus

1. Determining Even or Odd

Any even number modulo 2 equals 0. Any odd number modulo 2 equals 1. This is the most common use case in programming loops.

if num % 2 == 0:
  print("Even")

2. Extracting Right-most digits

Modulo 10 always strips the final digit off a number.

x = 4587 % 10
print(x) # Output: 7

9. Exponentiation ( ** )

Powers

The double asterisk mathematically raises the left operand to the power of the right operand.

Unlike languages that require Math.pow(), Python builds it directly into the syntax.

base = 4
power = 3

ans = base ** power # 4^3 = 4 * 4 * 4
print(ans)
# Output: 64

2. Relational (Comparison) Operators

Comparing values to yield Booleans.

12. Relational Operators Overview

Relational operators compare two operands. The final result of ANY relational operation is always a Boolean (True or False).

Symbol Name True if...
== Equal to The values of both operands are precisely equal.
!= Not equal to The values are different.
> Greater than Left is strictly greater than right.
< Less than Left is strictly less than right.
>= Greater than or equal Left is greater than OR equal to right.
<= Less than or equal Left is less than OR equal to right.

13. The Assignment ( = ) vs Equality ( == ) Trap

The #1 syntax error made by beginners.

Single Equal ( = )

ASSIGNMENT. This tells the computer: "Take the data on the right, and store it into the variable on the left."

x = 5

Double Equal ( == )

COMPARISON. This asks the computer a question: "Is the data on the left identical to the data on the right?"

x == 5 ➞ (Returns True/False)

14. Relational Examples in Action

Math Comparisons

x = 10
y = 20

print(x > y) # Output: False
print(x != y) # Output: True
print(x <= 10) # Output: True

String Comparisons

Strings are compared alphabetically (lexicographically) based on their ASCII values.

# 'Apple' comes before 'Banana'
a = "Apple"
b = "Banana"

print(a < b) # Output: True

15. ASCII Trap in String Comparison

Why capital letters matter.

In ASCII, all uppercase letters (A-Z) have lower numerical values than lowercase letters (a-z).

A = 65. a = 97.

res1 = "apple" == "Apple"
print(res1) # Output: False (Case sensitive)

res2 = "Zebra" < "apple"
print(res2) # Output: True (Because 'Z' is 90, 'a' is 97)

16. Advanced: Chaining Relational Operators

A powerful feature unique to Python is the ability to chain comparison operators together like in mathematics, without needing logical ANDs.

Standard Language Way

(Java, C++, JS)

x = 15
# (x > 10) AND (x < 20)

Python Way

Clean, readable, mathematical.

x = 15

ans = 10 < x < 20
print(ans) # Output: True

3. Logical Operators

Combining conditional statements.

17. Logical Operators Overview

Logical operators are used to connect multiple Boolean (True/False) expressions together.

and

Returns True ONLY if BOTH statements are true.

or

Returns True if AT LEAST ONE of the statements is true.

not

Reverses the result. Returns False if the result is true.

18. Truth Tables for AND / OR

OR Operator Table

19. Logical 'AND' & 'OR' in Action

Login System (AND)

Both the email and the password must match perfectly.

email_match = True
pass_match = False

login = email_match and pass_match
print(login) # Output: False

Coupon System (OR)

You get a discount if you have a coupon OR if it's your birthday.

has_coupon = False
is_birthday = True

discount = has_coupon or is_birthday
print(discount) # Output: True

20. Logical 'NOT' Operator

Flipping the Bit

The not operator evaluates a single boolean expression right after it, and flips the result completely. If it was True, it becomes False.

is_raining = True

# We want to go outside if it is NOT raining
go_outside = not is_raining

print(go_outside) # Output: False

Common Use Case:

Frequently used to check if a list or string is empty.

if not users:
  print("No users found")

4. Assignment Operators

21. The Core Assignment ( = )

We've already discussed this! It takes the R-value, evaluates it, and stores it inside the L-value container.

Multiple Assignment

Python allows you to assign multiple variables on an identical single line:

22. Augmented Assignment Operators

Shorthand for updating a variable.

In programming, it is incredibly common to update a variable based on its CURRENT value. For example, a video game score.

The Long Way

score = 10
# Add 5 points
score = score + 5

Python evaluates score + 5 (which is 15), then assigns 15 back overriding score.

The Augmented Way

score = 10
# Add 5 points
score += 5

This does exactly the same thing, but is faster to read and execute.

5. Identity Operators

is /// is not

24. Identity ( is / is not )

Comparing Memory Addresses, not Data Values.

The is operator returns True if both variables point to the exact same object in the computer's memory RAM. It does NOT just check if their data values look the same.

Identity ( is ) vs Equality ( == )

  • == asks: "Do these two boxes hold the same amount of money?"
  • is asks: "Are these two variables literally pointing to the EXACT SAME physical box?"

25. Identity Trap Example

Lists in Memory

Every time you create a new list with brackets `[ ]`, Python carves out a brand new memory block, even if the data inside is identical to another list.

x = ["apple", "banana"]
y = ["apple", "banana"]
z = x

# Do they have the same data?
print(x == y) # True

# Are they the exact same object?
print(x is y) # False! New memory block.
print(x is z) # True! Z points to X's block.

6. Membership Operators

in /// not in

26. Membership ( in / not in )

Definition

The in operator is used to test if a specific sequence is presented in an object (like a String, List, or Tuple). It returns True if it finds it.

This completely removes the need to write massive loops just to check if "Apple" is inside a grocery list!

fruits = ["apple", "banana", "mango"]

print("banana" in fruits)
# Output: True

print("cherry" in fruits)
# Output: False

print("cherry" not in fruits)
# Output: True

Precedence & Associativity

The order of operations.

27. What is Operator Precedence?

10 + 5 * 2 = ?

Is the answer 30? Or is the answer 20?

BODMAS / PEMDAS

Just like in mathematics, Python has strict rules on which operators get evaluated first when multiple are chained in the exact same line.

Because Multiplication * has higher precedence than Addition +, the 5 * 2 executes first.

ans = 10 + 5 * 2
# Python sees: 10 + (10)
print(ans) # Output: 20

28. The Precedence Table (Highest to Lowest)

Precedence Level Operator / Syntax Description
1 (Highest) (expressions) Parentheses (Always forced first)
2 ** Exponentiation
3 +x, -x, ~x Unary Plus, Unary Minus (Signs)
4 *, /, //, % Multiplication & both Divisions
5 +, - Addition & Subtraction
6 ==, !=, >, >=, <, <=, is, in Relational, Identity, Membership
7 not Logical NOT
8 and Logical AND
9 or Logical OR
10 (Lowest) =, +=, -=, *= All Assignment Operators

29. What is Associativity?

What happens when precedence ties?

Breaking the Tie

If two operators share the exact same precedence level (like * and /, which are both Level 4), Python uses Associativity to decide whether to evaluate Left-to-Right or Right-to-Left.

Left-to-Right (Almost Everything)

10 * 6 // 2
# 1. 10 * 6 = 60
# 2. 60 // 2 = 30

30. The Exception: Right-to-Left Associativity

Exponentiation ( ** )

Exponentiation evaluates from Right-to-Left.

2 ** 3 ** 2
# Evaluates (3 ** 2) first, which is 9.
# Then evaluates 2 ** 9.
# Output: 512

(If it was L-to-R, 2**3 = 8, 8**2 = 64).

Assignment ( = )

Assignment evaluates Right-to-Left, which makes perfect logical sense.

x = y = z = 10
# 1. z gets 10.
# 2. y gets z (10)
# 3. x gets y (10)

Mixed Calculations Breakdown

Analyzing complex expressions piece-by-piece.

31. Mixed Calc 1 (Easy Level)

ans = 5 + 3 * 2 == 11

Execution Order:

  1. Identify operators: +, *, ==
  2. Highest precedence: * ➔ (3 * 2 = 6)
  3. Next highest: + ➔ (5 + 6 = 11)
  4. Lowest precedence: == ➔ (11 == 11)
  5. Final Evaluation: True

Parenthesis Equivalent:

If Python automatically added brackets based on precedence, it would look like...

ans = ( (5 + (3 * 2)) == 11 )

32. Mixed Calc 2 (Medium Level)

ans = ( 10 // 3 ) ** 2 + 1 >= 10

Execution Order:

  • Step 1: Parentheses forced first: ( 10 // 3 ) evaluates to integer 3.
  • Expression is now: 3 ** 2 + 1 >= 10
  • Step 2: Exponentiation (highest left): 3 ** 2 evaluates to 9.
  • Expression is now: 9 + 1 >= 10
  • Step 3: Addition: 9 + 1 evaluates to 10.
  • Expression is now: 10 >= 10
  • Step 4: Relational: 10 is equal to 10. Final answer is True.

33. Mixed Calc 3 (Difficult Level)

ans = 15 % 4 is not 0 and "H" in "Hello"

Part A: 15 % 4 is not 0

1. Modulus: 15 / 4 is 3 with remainder 3.

2. Statement simplifies to: 3 is not 0

3. Identity check: True.

Part B: "H" in "Hello"

1. Membership check.

2. "H" does exist exactly in "Hello".

3. Evaluation: True.

Final Combination: True and True ➔ True

34. Shortcut Evaluation (Short-Circuit)

Logical AND Short-Circuit

If Python analyzes the left side of an and, and it evaluates to False, Python immediately skips evaluating the right side. It stops completely, saving processing time, because an AND can never be True if one part is already False.

Logical OR Short-Circuit

If Python analyzes the left side of an or, and it evaluates to True, Python immediately skips the right side, because an OR only needs one True to win.

x = 10 > 5 or (10 / 0 == 1)
# (10/0) would normally crash the program with a ZeroDivisionError!
# But because 10>5 is True, the 'or' operator short-circuits.
# The division (10/0) is literally never executed by the computer.

35. Complex Example 4 (Order of Execution)

val = 2 * 3 ** 2 + 5 // 2

Trace the Steps:

  • 1. Exponentiation has the highest precedence: 3 ** 2 == 9
  • Expression: 2 * 9 + 5 // 2
  • 2. Multiplication and Floor Division are equal (Level 4), so evaluate Left to Right.
  • 3. Leftmost is 2 * 9 which equals 18.
  • Expression: 18 + 5 // 2
  • 4. Next is 5 // 2 which equals 2 (2.5 rounded down).
  • Expression: 18 + 2
  • 5. Final Addition: 20.

36. Complex Example 5 (Strings & Logic)

word = "cat"
ans = (word * 2) == "catcat" and "dog" not in ["mouse", "fish"]

Analysis:

1. (word * 2) evaluates to "catcat".

2. "catcat" == "catcat" evaluates to True.

3. "dog" not in ["mouse", "fish"] evaluates to True.

4. True and True results in True.

37. Complex Example 6 (Identity Edge Cases)

Small Integer Caching in Python

Python optimizes memory by pre-allocating small integers (-5 to 256). If you assign two variables to 100, they will share the same memory address.

a = 100
b = 100
print(a is b) # Output: True

x = 1000
y = 1000
print(x is y) # Output: False

(Note: This behavior is specific to the standard CPython interpreter and can differ in REPL vs Script mode. This is why you should always use == for numbers, not is).

40. Final Advice for Developers

When in doubt, use Parentheses.

Even though Python has strict precedence rules, relying entirely on them can make your code very difficult for other human engineers to read.

Writing (5 + (2 * 4)) == 13 is physically identical to writing 5 + 2 * 4 == 13, but it communicates your exact intent instantly.
