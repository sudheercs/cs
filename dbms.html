An Introduction to Database Concepts

Understanding the Need for Databases

Presented by: Sudheer Singh

Date: 01/11/2025

What is Data?

Data refers to raw, unorganized facts. By itself it often lacks context or meaning.

Examples

  • Amit (a name)
  • 18 (a number)
  • New Delhi (a place)
  • 95.4 (a score)

Example of converting data into information:
“Amit, aged 18, from New Delhi, scored 95.4% in exams.”

Types of Data

  • Structured Data: Organized into tables (e.g., student records, sales data).
  • Semi-Structured Data: JSON, XML, where structure is flexible.
  • Unstructured Data: Images, videos, documents, or logs.

Example: Emails (semi-structured), customer reviews (unstructured).

What is a Database?

A database is a structured, organized collection of data stored and accessed electronically for efficient retrieval and management.

Real-World Examples

  • School Database: Students, teachers, grades, attendance.
  • Bank Database: Customers, transactions, accounts.
  • E-commerce Database: Products, users, orders, reviews.

Problems with Traditional File Systems

  • Data Redundancy: Same data repeated in multiple files.
  • Inconsistency: One file updated, others not.
  • Poor Data Sharing: Hard to access data simultaneously.
  • Security Problems: No control on who accesses what.
  • Integrity Issues: Hard to enforce business rules.

What is DBMS?

A Database Management System (DBMS) is software that helps create, maintain, and manipulate databases.

Functions of DBMS

  • Data storage, retrieval, and update.
  • User access control and security management.
  • Backup and recovery operations.
  • Concurrency control for multi-user systems.
  • Data integrity and consistency enforcement.

Examples of DBMS Software

  • Relational: MySQL, PostgreSQL, Oracle, SQL Server, SQLite.
  • NoSQL: MongoDB, Cassandra, Redis, Neo4j.
  • Cloud Databases: Amazon RDS, Google Cloud SQL, Firebase.

SQL Example:

SELECT * FROM Students WHERE Marks > 90;

Advantages of DBMS

  • Minimizes redundancy and inconsistency.
  • Improves data sharing and accessibility.
  • Provides centralized control and security.
  • Supports multi-user environment.
  • Ensures backup and recovery.
  • Maintains data integrity through constraints.

Relational Data Model

Introduced by Dr. E.F. Codd (1970). Data is stored in tables (relations) consisting of rows and columns.

Key Idea: Each table represents an entity (like Student, Course), and relationships are created using keys.

Example: Student Table

RollNoNameCourseAge
101AnkitCS18
102MeeraCommerce17
103RohanCS18
104PriyaScience17

Keys in DBMS

  • Primary Key: Uniquely identifies each record.
  • Foreign Key: References a key in another table.
  • Candidate Key: All possible unique identifiers.
  • Alternate Key: Candidate keys not chosen as primary.
  • Composite Key: Combination of two or more attributes.

Normalization

Process of organizing data to reduce redundancy and improve data integrity.

  • 1NF: Atomic values only.
  • 2NF: No partial dependency.
  • 3NF: No transitive dependency.

SQL Overview

  • DDL: CREATE, ALTER, DROP
  • DML: INSERT, UPDATE, DELETE
  • DCL: GRANT, REVOKE
  • TCL: COMMIT, ROLLBACK
CREATE TABLE Students(
  RollNo INT PRIMARY KEY,
  Name VARCHAR(50),
  Age INT,
  Course VARCHAR(30)
);

Database Users

  • Database Administrator (DBA): Manages and maintains database system.
  • Application Programmer: Develops software that interacts with DB.
  • End User: Uses applications to access data.

Summary

  • DBMS efficiently stores, retrieves, and manages data.
  • Relational model organizes data into tables.
  • Keys establish relationships and ensure uniqueness.
  • Normalization reduces redundancy.
  • SQL is the standard language for managing relational databases.
