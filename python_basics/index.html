1. Introduction to Python

The world's most versatile programming language.

What is it?

Python is a widely used general-purpose, high-level programming language.

Unlike languages built purely for web development (like JavaScript) or system programming (like C), Python is used for almost everything: Data Science, AI, Web Servers, and Automation.

Guido van Rossum

Python was created by Guido van Rossum in 1991 at the National Research Institute for Mathematics and Computer Science in the Netherlands.

It was named after the British comedy troupe Monty Python's Flying Circus, not the snake!

2. Core Features of Python (Part 1)

Easy to Learn and Use

Python is developer-friendly. It has a simple structure and clearly defined syntax.

It relies heavily on English keywords rather than punctuation, making it instantly readable for beginners.

Expressive Language

Python allows you to write programs with fewer lines of code compared to C++ or Java.

What might take 10 lines in Java can often be accomplished in a single, elegant line of Python.

3. Core Features of Python (Part 2)

Interpreted Language

Python does not need to be manually compiled into machine code before running.

The Python Interpreter processes the source code line by line at runtime, which makes debugging much faster and easier.

Cross-Platform / Portable

Python code is highly portable. A program written on a Windows machine will execute flawlessly on macOS or Linux without requiring any code modifications.

4. Core Features of Python (Part 3)

Free and Open Source

The Python language is freely available on python.org. It is managed by a vast open-source community that constantly improves its infrastructure.

Object-Oriented

Python perfectly incorporates Object-Oriented Programming (OOP) concepts: data encapsulation, inheritance, and polymorphism, allowing for complex architectural designs.

Large Standard Library

Python comes with "batteries included". The standard library is massive, providing out-of-the-box support for email, internet protocols, databases, and mathematics.

5. The "Hello World" Program

The traditional first step in any programming language.

In Python, displaying text on the screen requires exactly one line of code:

print("Hello, World!")

Compare this to Java:

class HelloWorld {
    public static void main(String[] args) {
        System.out.println("Hello, World!");
    }
}

6. Breaking down "Hello World"

print()

This is a built-in Function. A function is a reusable block of code designed to perform a specific action.

The job of the print() function is to take data and push it out to the console/screen.

"Hello, World!"

This is a String. A string is simply a sequence of characters acting as text data.

Python knows it is a string because it is strictly enclosed in double quotes ("") or single quotes ('').

7. Execution Modes

How do we actually run Python code?

Because Python is an interpreted language, it allows for flexibility in execution. There are two primary ways to interact with the Python interpreter:

1. Interactive Mode

A live command-line environment (REPL) where you type a single command and Python instantly executes it.

2. Script Mode

Writing an entire program into a file (ending in .py) and then telling Python to execute the whole file at once.

8. Interactive Mode

The REPL (Read, Evaluate, Print, Loop)

The Python Prompt

When you start Python in a terminal, you are greeted by three chevrons: >>>. This is the prompt.

When you type an expression and hit Enter, Python immediately gives the output.

# Output in Terminal: >>> 5 + 5 10 >>> print("Testing") Testing >>> x = 20 >>> x * 2 40

9. Pros and Cons of Interactive Mode

✅ Advantages

  • Instant Feedback: Excellent for beginners testing out how a specific function works.
  • Great Calculator: Perfect for rapidly testing math equations or one-line logic blocks.

❌ Disadvantages

  • No Saving: Once you close the terminal, your code is gone forever.
  • Tedious for large apps: You cannot easily write a 100-line looping program in an interactive prompt without getting confused.

10. Script Mode

How professional software is written.

Writing the Script

You use a text editor (like Notepad, VSCode, or IDLE) and type all your code at once.

You save the file with a .py extension (e.g., game.py).

Running the Script

You pass the file to the interpreter via the command line.

C:\Users\John> python game.py

Python will then read the file from top to bottom, executing every line silently unless instructed to print.

11. Pros and Cons of Script Mode

✅ Advantages

  • Permanent: Code is saved permanently on your hard drive.
  • Editable: You can quickly fix a mistake on line 50 without retyping lines 1-49.
  • Scalable: Necessary for apps, games, and web servers.

❌ Disadvantages

  • No Auto-Print: You don't see intermediate outputs unless you explicitly write print() statements, unlike the interactive prompt which auto-prints results.
  • Slightly more setup required to run a simple math test.

12. Python Character Set

The alphabet of the Python language.

What is a Character Set?

A Character Set is a valid set of characters that a language can recognize. If you type a character outside of this set, the Python interpreter will throw a Syntax Error.

Python fully supports the Unicode standard, meaning it can technically process text in global languages. However, the syntax of the language itself relies on a core set of standard characters.

13. Core Character Set Breakdown

🔤 Letters

Uppercase A-Z and Lowercase a-z.

Note: Python is case-sensitive! (A is not a)

🔢 Digits

All numbers 0 through 9.

✨ Special Symbols

Space, + - * / ( ) [ ] { } = @ _ , . ; : ' " \ % #

These have specific syntactic meanings.

💨 Whitespace

Blank Spaces, Tabs, Carriage Returns, and Newlines. Whitespace is critical in Python for defining code blocks.

14. Python Tokens (Lexical Units)

The atoms of a program.

What is a Token?

In English, the smallest individual unit in a sentence is a word or a punctuation mark. In Python, the smallest individual unit in a program is known as a Token.

The 5 types of Tokens in Python:

  1. Keywords
  2. Identifiers
  3. Literals
  4. Operators
  5. Punctuators

15. Token 1: Keywords

Reserved words with special meaning.

What are they?

Keywords are predefined, reserved words used by the Python compiler.

They dictate the structure and logic of the code. You cannot use a keyword as a variable name or a function name.

Examples

False
True
None
and
as
break
class
def
elif
else
for
if

16. Token 2: Identifiers

Names given by the programmer to entities.

What are they?

An Identifier is a fundamental building block. It is a name given to parts of a program, like variables, functions, objects, or classes.

While keywords are defined by Python, identifiers are created and defined by you.

# 'age' and 'salary' are Identifiers
age = 25
salary = 50000.0

17. Rules for Naming Identifiers

✅ Valid Rules

  • Must start with an alphabet letter (A-Z, a-z) or an underscore ( _ ).
  • Can be followed by letters, digits, or underscores.
  • They are case-sensitive. (Age and age are two entirely different identifiers).

❌ Invalid Rules

  • Cannot contain any special characters (like !, @, #, $, %).
  • Cannot contain blank spaces.
  • Cannot start with a digit! (e.g., 1st_Name is illegal).
  • Cannot be a reserved keyword.

18. Token 3: Literals (Values)

Constant, fixed data items.

What is a Literal?

A Literal is the raw data that you assign to an identifier. It is a value that literally represents itself and does not change during execution.

Example

x = 100
name = "Alice"

Here, 100 and "Alice" are literals. x and name are identifiers.

19. Common Types of Literals

String Literals

Text enclosed in quotes.
"Hello" or 'Hello'.

Numeric Literals

Integers (14, -5)
Floats (3.14, 0.0)
Complex (3+4j)

Boolean Literals

Exactly two values representing truth:
True and False (Must be capitalized).

Special Literal: None

None is used to signify the absence of value or 'nothingness' in a variable.

20. Token 4: Operators

Symbols that trigger computation.

Operators are special symbols that carry out arithmetic or logical computation on values or variables (known as Operands).

Type Symbols Example
Arithmetic +, -, *, /, % (modulo), ** (exponent) 5 * 2 = 10
Relational ==, !=, >, <, >=, <= 4 > 3 -> True
Logical and, or, not True and False
Assignment =, +=, -=, *= x = 5

21. Token 5: Punctuators (Delimiters)

What are they?

Punctuators are symbols that are used to organize programming-sentence structures, and indicate the rhythm and emphasis of expressions, statements, and program flow.

Examples in Python

  • Parentheses: ( )
  • Brackets: [ ]
  • Braces: { }
  • Comma: ,
  • Colon: : (used to start blocks)
  • Quote: " "

22. Understanding Variables

Containers for storing data values.

What is a Variable?

A Variable is a named location in the computer's memory (RAM) used to store data.

Wait, isn't that an Identifier? Yes! A variable is specifically an identifier that points to a memory address holding a literal value.

Creating a Variable

Unlike languages like C or Java, Python has no command for declaring a variable upfront. A variable is automatically created the moment you first assign a value to it.

x = 5
y = "John"

23. Python is Dynamically Typed

No Type Boundaries

In languages like C++, if you declare an integer variable, it can ONLY ever hold integers. If you try to store a string in it later, the program crashes.

Python variables do not need to be declared with any particular type, and can even change type dynamically after they have been set.

x = 4 # x is an integer
x = "Sally" # x is now a string
print(x)

Output: Sally

24. Concept of L-value and R-value

Understanding the mechanics of the assignment operator (=).

A = B

The assignment operator = divides an expression into a Left side and a Right side.

L-value (Left Value)

An expression that refers to a memory location. It must represent a container that can actually hold data. Variables are L-values.

R-value (Right Value)

An expression that refers to the data value that is stored at some memory address. Literals and constants are strictly R-values.

25. L-value vs R-value Examples

✅ Valid Statement

X = 20

• X is a valid L-value (it's a container in memory).
• 20 is a valid R-value (it's the data).

❌ Invalid Statement

20 = X

• 20 is a literal number, not a memory container. It cannot be an L-value. This will throw a SyntaxError.

✅ Variables as R-values

A = 10
B = A

In B = A, B acts as the L-value (container), and A acts as the R-value (we extract the data value 10 out of container A and put it into container B).

26. Use of Comments

Communicating with humans, not the computer.

What are Comments?

Comments are text notes added to the code to provide explanatory information. They act as internal documentation.

When the Python interpreter runs the program, it completely ignores any text that is marked as a comment.

Why do we need them?

  • To clarify the intent behind complex logic for other developers.
  • To remind yourself what your code does when you look at it 6 months later.
  • To temporarily disable lines of code while debugging (known as "commenting out").

27. How to write Comments in Python

1. Single-Line Comments ( # )

Created by using the hash/pound symbol. Everything to the right of the # on that line is ignored.

# Calculate the area of a circle
area = 3.14 * (radius ** 2)

print(area) # This prints the result to the screen

28. Multi-Line Comments (Docstrings)

2. Multi-Line Comments ( """ )

If you have a massive paragraph to write, prepending # to every single line is tedious. Python allows you to use triple quotes to write across multiple lines.

"""
This is a multi-line comment.
It spans across three different lines.
The interpreter will ignore all of this.
"""
def my_function():
    pass

(Note: Technically these are regular strings that are just not assigned to a variable, so they are garbage-collected immediately, acting as comments).

29. Summary of Python Basics

  • Python: A high-level, interpreted, easy-to-read language.
  • Modes: Interactive (immediate feedback) vs Script (permanent file).
  • Tokens (The 5 Atoms): Keywords, Identifiers, Literals, Operators, Punctuators.
  • Variables: Dynamically typed memory locations.
  • L-value vs R-value: L-value gets the assignment (container), R-value is the data provided.
  • Comments: Notes for humans, ignored by the Python interpreter.
